# Запити зацікавлених осіб

## Вступ

Система управління відкритими даними забезпечує створення, зберігання, оновлення та пошук інформації в базах даних, контролюючи доступ до даних. Оскільки ці процеси є складними, документ "Запити зацікавлених сторін" необхідний для надання учасникам проєкту чіткого розуміння принципів їх функціонування в межах системи.

### Мета

Цей документ має на меті сформувати загальне уявлення про продукт, проаналізувати ключові бізнес-процеси, що відбуваються в системі, а також визначити їхніх учасників та їхні рівні доступу. Крім того, він слугує для оцінки продукту, акцентуючи увагу на його сильних і слабких сторонах за різними критеріями.

### Контекст

Цей документ стосується проєктів, спрямованих на розробку та вдосконалення платформ для зберігання, обробки й надання відкритих даних громадськості та бізнесу. Він впливає на впровадження систем управління даними, які забезпечують їхню доступність, прозорість і ефективне використання. Такі дані можуть застосовуватися для аналітики, наукових досліджень, державного управління та створення нових цифрових продуктів і послуг.


### Основні визначення та скорочення

[Дані](https://uk.wikipedia.org/wiki/%D0%94%D0%B0%D0%BD%D1%96) (Data) – це набір фактів, числових значень або інших інформаційних одиниць, які використовуються для аналізу, ухвалення рішень або створення моделей. Вони можуть бути представлені у вигляді чисел, тексту, зображень або інших форматів та зазвичай проходять обробку чи аналіз для отримання корисної інформації чи висновків.

[Таблиця](https://uk.wikipedia.org/wiki/%D0%A2%D0%B0%D0%B1%D0%BB%D0%B8%D1%86%D1%8F_(%D0%B1%D0%B0%D0%B7%D0%B8_%D0%B4%D0%B0%D0%BD%D0%B8%D1%85)) (Table) – це основний структурний елемент бази даних, що організовує інформацію у формі рядків і стовпців. Кожен рядок відповідає окремому запису, а кожен стовпець – його атрибуту чи характеристиці. Таблиці є ключовим засобом збереження даних у реляційних базах, надаючи можливість впорядкованого зберігання інформації.

[Запит](https://www.kievoit.ippo.kubg.edu.ua/kievoit/2012/25/25.html) (Query) – це команда, що використовується для отримання певної інформації з бази даних. Запити формулюються за допомогою спеціалізованих мов, таких як SQL (Structured Query Language), і можуть включати фільтрацію, сортування та інші умови вибірки необхідних даних.

[CSV](https://uk.wikipedia.org/wiki/CSV) (Comma Separated Values) – це текстовий формат файлу, що використовується для збереження табличних даних, де значення у рядку розділені комами. Формат CSV широко застосовується для зберігання відкритих даних, оскільки його легко імпортувати у програми, такі як Excel, і швидко обробляти.

[JSON](https://uk.wikipedia.org/wiki/JSON) (JavaScript Object Notation) – це компактний формат обміну даними, що базується на тексті. Його часто використовують для збереження та передачі структурованої інформації між веб-сервісами та клієнтськими додатками. JSON є популярним у веб-розробці завдяки простоті та зручності у використанні.

[SQL](https://uk.wikipedia.org/wiki/SQL) (Structured Query Language) – це мова програмування для роботи з реляційними базами даних. Вона дозволяє виконувати операції створення, читання, оновлення та видалення даних. За допомогою SQL можна вибирати необхідні дані, застосовувати фільтри чи об’єднувати кілька таблиць для отримання складних результатів.

[NoSQL](https://uk.wikipedia.org/wiki/NoSQL) (Not Only SQL) – це категорія баз даних, призначених для зберігання великих обсягів неструктурованої або частково структурованої інформації. Відрізняється гнучкістю у моделюванні даних, що робить її зручною для роботи з великими та динамічними наборами інформації, такими як соціальні мережі, інтернет-логи або онлайн-магазини.

[Великий обсяг даних](https://uk.wikipedia.org/wiki/%D0%92%D0%B5%D0%BB%D0%B8%D0%BA%D1%96_%D0%B4%D0%B0%D0%BD%D1%96) (Big Data) — це термін, який описує величезні набори даних, що характеризуються трьома основними ознаками: великим обсягом, високою швидкістю створення та великою різноманітністю. Ці дані часто надходять у нерегулярній формі (незалишкові), з багатьох джерел і в різних форматах (текстові файли, мультимедіа, лог-файли). Обробка таких даних вимагає використання спеціалізованих методів та інструментів (наприклад, Hadoop, Spark), оскільки традиційні бази даних і програми не здатні ефективно обробляти настільки великі та складні обсяги інформації.

[Стандартизація даних](https://profisee.com/blog/what-is-data-standardization) (Data Standardization) – це процес уніфікації даних для забезпечення їх узгодженості та легкості аналізу. Він є особливо важливим при зборі інформації з різних джерел, де можуть використовуватися різні формати або одиниці вимірювання. Наприклад, це може включати уніфікацію форматів дат або приведення грошових значень до єдиної валюти.

[Анонімізація даних](https://en.wikipedia.org/wiki/Data_anonymization) (Data Anonymization) – це метод обробки інформації, що приховує або змінює персональні дані для запобігання ідентифікації окремих осіб. Це важливий процес у сферах охорони здоров'я, фінансів та соціальних послуг, де захист конфіденційної інформації є критичним.

[Візуалізація даних](https://en.wikipedia.org/wiki/Data_and_information_visualization) (Data Visualization) – це спосіб представлення інформації у вигляді графіків, діаграм або інтерактивних карт. Основна мета – спростити аналіз великих обсягів даних і виявити приховані закономірності або тренди, які важко побачити в числових таблицях. Візуалізація відіграє важливу роль у розумінні складних статистичних показників, таких як економічні тенденції або медичні дослідження.

## <span id="descr">Короткий огляд продукту</span>

Система управління відкритими даними (Open Data Management System) – це програмне забезпечення, яке надає широкий набір інструментів для збору, збереження, обробки та публікації даних у відкритий доступ. Вона допомагає організаціям ефективно керувати великими масивами інформації, отриманими з різних джерел, таких як API, бази даних та файли у форматах CSV, JSON тощо.

Система містить функції для стандартизації та очищення даних, що сприяє підвищенню їхньої точності та якості. Вона підтримує автоматичне оновлення інформації, а також надає можливість публікації даних на відкритих платформах або через веб-інтерфейси. Вбудовані інструменти візуалізації дозволяють легко аналізувати інформацію за допомогою графіків, діаграм та інших наочних форм представлення.

Гнучка система керування доступом забезпечує безпечне зберігання конфіденційної інформації та контроль над її використанням. Завдяки цьому рішення сприяє прозорості, підвищує відкритість урядових і комерційних даних, а також покращує доступ до інформації для громадськості, дослідників і бізнесу.



## <span id="functionality">Функціональність</span>

### <span id="user-func">Функції, доступні користувачам:</span>

1. Пошук і перегляд інформації – користувачі можуть знаходити відкриті дані, здійснюючи пошук за категоріями, темами або ключовими словами, а також переглядати доступні набори даних.

2. Завантаження наборів даних – можливість отримувати потрібні дані у різних форматах (CSV, JSON, XML) для подальшого аналізу чи використання.

3. Фільтрація та впорядкування – користувачі можуть застосовувати фільтри для вибору релевантних даних, а також сортувати їх за певними критеріями.

4. Графічне представлення інформації – доступ до інструментів візуалізації, що дозволяють створювати графіки, діаграми та інтерактивні карти на основі вибраних даних.

5. API-доступ – можливість отримання даних через API для інтеграції з іншими програмними продуктами та аналітичними системами.

6. Збереження запитів і результатів – користувачі можуть зберігати створені запити та отримані результати для швидкого доступу або повторного використання.

7. Залишення відгуків – можливість коментувати та оцінювати якість наборів даних, а також повідомляти про помилки для їхнього вдосконалення.

### <span id="admin-func">Функції, доступні адміністраторам:</span>

1. Керування джерелами даних – адміністратори можуть додавати, редагувати або видаляти джерела інформації, а також налаштовувати автоматичне оновлення даних із зовнішніх систем, таких як API чи бази даних.

2. Очищення та уніфікація даних – функціонал для видалення дублікатів, виправлення помилок і приведення інформації до єдиного формату перед її публікацією.

3. Розподіл прав доступу – можливість налаштовувати рівні доступу для різних категорій користувачів, обмежувати перегляд конфіденційних даних або надавати доступ лише до певної інформації.

4. Моніторинг використання – відстеження активності користувачів, історії запитів до даних і API, а також генерація звітів щодо роботи системи.

5. Керування публікацією – адміністратори можуть контролювати випуск нових наборів даних, редагувати або видаляти застарілу інформацію та визначати частоту її оновлення.

6. Інтеграція з іншими платформами – можливість підключення до внутрішніх або зовнішніх систем через API або спеціалізовані інтеграційні модулі.

7. Безпека та аудит – інструменти для контролю безпеки системи, впровадження політик захисту даних та проведення аудиту для виявлення аномальної активності або потенційних загроз.
